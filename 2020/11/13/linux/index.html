linux介绍

linux是一个开源、免费的操作系统，是一个基于POSIX和UNIX的多用户、多任务、支持多线程和多CPU的操作系统，目前，很多大型项目都部署在linux上，在linux中，一切皆文件

linux目前的发行版本主要有：CentOs、Ubuntu、redhat

由于没有安装linux虚拟机，所以要借助工具远程连接服务器，需要借助以下工具：

  1. 宝塔面板

  2. xshell + xftp（finalshell + forklift）
    如果在windows环境下：可以使用 xshell + xftp 远程控制服务器和传输文件
    mac下：可以使用 finalshell + forklift 代替win下的 xshell + xftp

文件/目录操作

查找文件

  • 按文件名查找：find /路径 -name '文件名'

  • 按文件特征查找：find /路径 特征
    特征主要有：

    • -amin -time：在time分钟时间内访问的文件

    • -atime -time：在time小时内访问的文件

    • -empty：空的文件

    • -mmin -time：time分钟内修改过的文件

    • -mtime -time：time小时内修改过的文件

    • -user -username：属于username这个用户的文件

    • -size +size[c:字节,k:kb,M:Mb,G:GB]：大于size的文件

    • -size -size：小于size的文件

  • 混合查找：! -and -or

查看目录下的文件

ls /

ls -a：查看全部文件，包括隐藏文件

ls -l：列出除了隐藏文件以外的文件，包括文件的属性和权限

ls -al：列出全部文件及属性和权限

以下是对这些目录的解释：

目录 说明
/bin bin是Binary的缩写, 这个目录存放着最经常使用的命令。
/boot 这里存放的是启动Linux时使用的一些核心文件，包括一些连接文件以及镜像文件。
/dev dev是Device(设备)的缩写, 存放的是Linux的外部设备，在Linux中访问设备的方式和访问文件的方式是相同的。
/etc 这个目录用来存放所有的系统管理所需要的配置文件和子目录。
/home 用户的主目录，在Linux中，每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的。
/lib 这个目录里存放着系统最基本的动态连接共享库，其作用类似于Windows里的DLL文件。
/lost+found 这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件。
/media linux系统会自动识别一些设备，例如U盘、光驱等等，当识别后，linux会把识别的设备挂载到这个目录下。
/mnt 系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将光驱挂载在/mnt/上，然后进入该目录就可以查看光驱里的内容了。
/opt 这是给主机额外安装软件所摆放的目录。比如你安装一个ORACLE数据库则就可以放到这个目录下。默认是空的。
/proc 这个目录是一个虚拟的目录，它是系统内存的映射，我们可以通过直接访问这个目录来获取系统信息。
/root 该目录为系统管理员，也称作超级权限者的用户主目录。
/sbin s就是Super User的意思，这里存放的是系统管理员使用的系统管理程序。
/srv 该目录存放一些服务启动之后需要提取的数据。
/sys 这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统 sysfs 。
/tmp 这个目录是用来存放一些临时文件的。
/usr 这是一个非常重要的目录，用户的很多应用程序和文件都放在这个目录下，类似于windows下的program files目录。
/usr/bin 系统用户使用的应用程序。
/usr/sbin 超级用户使用的比较高级的管理程序和系统守护程序。
/usr/src 内核源代码默认的放置目录。
/var 这个目录中存放着在不断扩充着的东西，我们习惯将那些经常被修改的目录放在这个目录下。包括各种日志文件。
/run 是一个临时文件系统，存储系统启动以来的信息。当系统重启时，这个目录下的文件应该被删掉或清除。

文件基本属性操作

Linux系统是一种典型的多用户系统，不同的用户处于不同的地位，拥有不同的权限。为了保护系统的安全性，Linux系统对不同的用户访问同一文件（包括目录文件）的权限做了不同的规定。

文件的基本属性

在Linux中第一个字符代表这个文件是目录、文件或链接文件等等：

当为[ d ]则是目录

当为[ - ]则是文件；

若是[ l ]则表示为链接文档 ( link file )；

若是[ b ]则表示为装置文件里面的可供储存的接口设备 ( 可随机存取装置 )；

若是[ c ]则表示为装置文件里面的串行端口设备，例如键盘、鼠标 ( 一次性读取装置 )。

接下来的字符中，以三个为一组，且均为『rwx』 的三个参数的组合。

其中，[ r ]代表可读(read)、[ w ]代表可写(write)、[ x ]代表可执行(execute)。

要注意的是，这三个权限的位置不会改变，如果没有权限，就会出现减号[ - ]而已。

对于文件来说，它都有一个特定的所有者，也就是对该文件具有所有权的用户。

同时，在Linux系统中，用户是按组分类的，一个用户属于一个或多个组。

文件所有者以外的用户又可以分为文件所有者的同组用户和其他用户。

因此，Linux系统按文件所有者、文件所有者同组用户和其他用户来规定了不同的文件访问权限。

十位字母后的数字属性表示 连接数：有多少个文件名连接到这个文件

数字后的两个属性分别表示 属主 和 属组（用户组）

再后面是文件大小 和 创建时间

修改文件属性

修改文件属组

chgrp -R 属组名 要改的文件名

chgrp -R 属主名:属组名 要改的文件名

修改文件权限

chmod [-R] rwx 文件/目录

Linux文件的基本权限就有九个，分别是owner/group/others三种身份各有自己的read/write/execute权限。

文件的权限字符为：『-rwxrwxrwx』，这九个权限是三个三个一组的！其中，我们可以使用数字来代表各个权限，各权限的分数对照表如下：

r：4 w：2 e：1 -：0

如：

chmod 777 filename： 赋予最高权限

chmod 770 filename： 其他用户没有权限

查看文件内容

Linux系统中使用以下命令来查看文件的内容：

  • cat 由第一行开始显示文件内容

  • tac 从最后一行开始显示，可以看出 tac 是 cat 的倒着写！

  • nl 显示的时候，顺道输出行号！

  • more 一页一页的显示文件内容，空格翻页，回车下一行

  • less 与 more 类似，但是比 more 更好的是，他可以往前翻页！看完按q退出
    在less中，方向键上下移动，用/向下查找字符串，?向上查找，n表示下一个，N表示上一个

  • head -n 10 只看前10行

  • tail -n 10 只看尾10行

硬链接和软链接

linux中链接分为两种：

  • 硬链接
    硬链接表示一个文件可以拥有多个访问它的路径（拥有多个备份），修改任意一个文件，其他文件也会自动被修改，相当于指针，通常使用硬链接对文件进行备份，防止文件被误删

  • 软链接
    软链接类似window下的快捷方式，删除源文件，就不能通过访问快捷方式访访问源文件了

创建硬链接：ln f1 f2 表示创建f1的硬链接f2

创建软链接：ln -s f1 f 表示创建f1的软链接f3

vim编辑器的使用

vim是一个文本编辑器，在vim中，有三种使用模式：

  • 命令模式
    启动vim，便进入命令模式
    在命令模式下，键盘输入数字+空格：跳转到当前光标下的第n个下标
    H：移动到文本内容第一个字符
    M：移动到中间字符
    L：移动到最后一个字符
    /：向下寻找字符(串)
    ?：向上寻找

  • 输入模式

输入i、a、或o，进入插入模式，输入完毕，按esc回到命令模式

  • 底线命令模式
    输入 :wq 保存，结束底线命令模式
    其中w为保存，q为退出

还有其他一些底线命令：

  • w!：强制保存

  • q!：强制退出

  • ZZ：如果未编辑则不保存退出，如果编辑了则保存退出

  • set nu：设置行号

  • set nonu：取消行号

账号管理

Linux系统是一个多用户多任务的分时操作系统，任何一个要使用系统资源的用户，都必须首先向系统管理员申请一个账号，然后以这个账号的身份进入系统。

实现用户账号的管理，要完成的工作主要有如下几个方面：

  • 用户账号的增删改

  • 用户密码的管理。

  • 用户组的管理。

  • 添加用户
    添加用户：useradd -可选项 用户名
    可选项：

    • -c comment 指定一段注释性描述。

    • -d 目录 指定用户主目录，如果此目录不存在，则同时使用-m选项，可以创建主目录。

    • -g 用户组 指定用户所属的用户组。

    • -G 用户组，用户组 指定用户所属的附加组。

    • -m　使用者目录如不存在则自动建立。

    • -s Shell文件 指定用户的登录Shell。

    • -u 用户号 指定用户的用户号，如果同时有-o选项，则可以重复使用其他用户的标识号。

  • 删除用户
    删除用户：userdel -r 用户名

-r 表示层级删除

  • 修改用户
    usermod 可选项 用户名，其中可选项和增加用户的可选项一致

  • 切换用户
    su 用户名 $表示普通用户
    可以使用再次使用su命令切换回原来的用户，也可以使用exit直接退出

  • 赋予用户密码
    passwd 用户名
    只有root用户才可以赋予密码

  • 锁定用户
    如果你的用户权限比较高，还可以锁定别人的用户
    锁定用户：passwd -l 用户名
    解锁：passwd -u 用户名
    删除用户密码：passwd -d 用户名

对用户账号的管理，实质上是对/etc/passwd文件进行修改

/etc/passwd中一行记录对应着一个用户，每行记录又被冒号(:)分隔为7个字段，其格式和具体含义如下：

用户名:密码(x表示不可见):用户标识号:组标识号:注释性描述:主目录:登录Shell

真正加密后的密码是在/etc/shadow文件中

用户组管理

每个用户都有一个用户组，系统可以对一个用户组中的所有用户进行集中管理。

用户组的管理涉及用户组的添加、删除和修改。组的增加、删除和修改实际上就是对/etc/group文件的更新。

  • 创建用户组
    groupadd 组名
    创建完用户组之后可以得到一个组的id，这个id是可以指定的，这个id表示权限的大小
    groupadd -g id

  • 删除用户组
    groupdel 组名

  • 修改用户组信息
    groupmod [-g -n] 组名
    groupmod -g 520 -n leo leobro：将leobro组的id修改为520，将组名修改为bro

  • 切换组
    newgrp 组名

磁盘管理

df -h查看系统磁盘总体情况

du -h查看当前目录的使用情况

检查根目录下每个目录的情况：du -sm /*

在mac或linux下，如果想要挂载外部设备，使用命令mount /dev/外部设备 /mnt/目录 将外部设备挂载到mnt的目录下

卸载：unmount -f(强制)

进程管理

  • 查看当前系统进程的信息
    ps -xx：
    • ps -a：显示终端运行的当前进程信息
    • ps -u：以用户的信息显示进程
    • ps -x：显示后台运行进程的参数

ps -aux：综合以上三个

使用 ps -aux|grep xxx 查看与xxx相关的所有进程

ps -ef：查看父进程信息

pstree -pu：以树的方式显示父id和用户组

lsof -i 端口号：查看某个进程

kill -9 进程id：强制结束进程

版本控制

版本控制是一种在开发过程中用于管理我们对文件、目录或工程等内容的修改历史，方便查看更改历史记录，备份以便恢复以前的版本的软件工程技术

版本控制技术用于管理多人协同开发项目，使用版本控制技术能实现多人协同开发，提高开发效率

目前主要的版本控制器有：

  • git

  • svn

  • cvs

  • vss

  • tfs

最广泛使用的是git和svn

版本控制分类

  • 本地版本控制

  • 集中式版本控制 svn

  • 分布式版本控制 git

所有版本信息仓库全部同步到本地的每个用户，每个用户都可以在本地查看所有的版本历史

git的环境配置

查看git的安装目录：which git

设置用户名和邮箱：

查看git配置：git config -l

查看系统配置：git config --system --list

查看用户配置：git config --global --list

.gitconfig文件：

gitconfig文件内容与终端查看的内容一样

git的基本理论

git本地有三个工作区域：

  • workspace：工作区

平时存放项目代码的地方

  • stage/index：暂存区

临时存放对代码的改动，事实上只是一个文件，保存时将提交到文件列表信息中

  • 资源库

本地安全存放数据的位置，里面有提交所有版本的数据，head指向最新放入仓库的版本

  • 远程仓库

git的工作流程：

先通过git add files将数据从工作区添加到暂存区，再通过git commit提交到本地git仓库，最后通过git push添加到远程仓库

搭建git项目

创建本地仓库：git init

创建后，就会自动新建一个隐藏文件.git

克隆远程仓库：git clone url

git的基本操作命令

查看文件状态

查看指定文件状态：git status [filename]

查看所有文件状态：git status

添加所有的文件到暂存区

git add .

提交暂存区的内容到本地仓库

git commit -m "message"

忽略文件

在主目录下建立.gitignore文件，该文件的规则如下：

  • 空行或者以 # 号开始的行会被忽略

  • *.text 忽略所有.text结尾的文件

  • ！表示例外规则，不会被忽略

  • /temp 不忽略temp目录

  • temp/ 忽略temp目录下的文件

  • temp/*.text 忽略temp下所有text文件，但不忽略temp子目录下的文件

git分支

分支可以理解为允许的环境

查看所有分支：git branch

查看远程分支：git branch -r

新建分支：git branch xxx

新建远程分支：git push --set-upstream origin dev

但当前分支仍然是主分支master

新建分支并切换分支：git checkout -b xxx

切换分支：git checkout dev

合并当前分支到指定分支：git merge [branch_name]

删除分支：git branch -d[branch_name]

删除远程分支：git branch -dr[

多人协作下，如果修改了同一个文件，就会产生冲突，这时候要 修改冲突文件，选择其中一个修改的版本，再次提交。

master主分支必须非常稳定，用来发布新的版本，发布新版本时，只要把其他分支merge到主分支就可以了
